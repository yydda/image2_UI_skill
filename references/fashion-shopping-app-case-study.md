# 女装购物 App 案例拆解

这个案例适合演示 `moni-ui-skill` 处理“参考设计风格 + 明确要求生成展示海报 / App mockup”时的判断方式：核心目标不是先写结构代码，而是先识别必须真实生图的高影响视觉资产，再决定哪些区域继续用代码承载。

## 适用场景

当用户提出这类请求时，优先参考这个案例：

- “参考这个风格，生成一个女装购物 App。”
- “要 16:9 展示海报 / 官网首屏 mockup。”
- “适配手机 App 观看的双设备展示图。”
- “这次必须按参考图风格真实生图，不要只做代码近似。”

这个案例覆盖的不是单纯移动端页面复刻，而是“面向展示与提案的 App 视觉稿”：

- 左侧手机强调品牌导购与氛围首屏。
- 右侧手机强调购物首页、商品卡片与营销模块。
- 外围还有包、花瓶、高跟鞋、丝绸布料等陈列型视觉元素。

## 这个案例教什么

### 1. 用户要的是展示级视觉稿，不只是可运行页面

这个案例里，用户需求已经把目标说得很清楚：

- 参考现有风格。
- 生成一张 16:9 比例的女装购物 App 展示海报。
- 重点保留柔和粉紫配色、双手机构图和时尚电商质感。

所以任务重心应该先落在：

- 双手机 mockup 构图。
- 主服装图、模特图、商品图、包和鞋等真实视觉素材。
- 统一的柔粉色系、布光、材质和陈列氛围。

而不是上来先做 HTML/CSS 骨架再声称“已经完成 image_gen”。

### 2. 这个案例默认应触发真实生图优先

这类时尚电商展示图里，以下区域如果不真实生图，质感会明显掉：

- 首屏服装主视觉。
- 商品卡中的模特图、服装图、包袋图。
- 场景化装饰物，如高跟鞋、花瓶、丝绸布料、陈列台。
- 双手机外部的大面积氛围背景与柔光。

因此这个案例应明确归类为：

- **必须真实生图**：主视觉服装、模特、配饰、陈列型静物、海报级背景氛围。
- **必须代码实现**：按钮、搜索框、价格、标题、标签、导航、图标、列表结构。
- **可混合处理**：手机外壳、部分圆形色块、简单底部导航容器。

### 3. “App 页面”与“展示海报”可以同时存在

这个案例很适合教模型不要二选一：

- 手机内部页面内容仍然应该尽量可拆成真实 UI 结构。
- 但展示图整体仍需要真实位图主导视觉效果。

正确理解是：

- 页面内部的文字、价格、导航、搜索框优先保留为代码语义。
- 页面内部的大面积服装图、模特图、包袋图等由 image_gen 提供真实位图。
- 外围静物陈列和整体背景氛围也应通过 image_gen 完成。

## 前期审查示例

### 整体判断

- 页面类型：女装购物 App 展示海报 / 双手机营销视觉稿。
- 视觉风格：柔粉、浅紫、奶油白、轻奢女性向、编辑风陈列摄影。
- 核心布局：双手机并排 + 手机外部静物陈列 + 低饱和柔光背景。
- 首屏重点：左屏主服装视觉、右屏购物首页、底部电商入口和氛围配饰。
- 主要风险：多张商品图风格一致性、粉色调过饱和、人物和服装质感不统一、文字被错误烘焙进图片。

### 元素拆分

| 区域/元素 | 建议实现方式 | 难度 | 原因 | 是否需要用户确认 |
| --- | --- | --- | --- | --- |
| 双手机布局和基础壳体 | 代码 + 少量位图 | 中等 | 轮廓可代码实现，但展示稿需高质感整合 | 否 |
| 搜索框、价格、导航、icon | 代码 | 容易 | 必须保持可编辑和可访问 | 否 |
| 左屏主服装图 | image_gen | 困难 | 服装版型、面料光泽和时尚感高度依赖真实位图 | 否 |
| 右屏商品卡缩略图 | image_gen | 困难 | 多商品图需要统一风格和光照 | 否 |
| 包袋、高跟鞋、花瓶、布料 | image_gen | 困难 | 属于陈列型商品摄影 / 静物场景 | 否 |
| 背景柔光和粉紫氛围 | CSS + image_gen | 中等 | 可混合实现，但海报级效果更适合用真实位图打底 | 否 |
| 品牌名和营销文案 | 代码 | 容易 | 可编辑文本，不能烘焙进图里 | 否 |
| 标题字体气质 | 授权字体或开源近似 | 中等 | 高反差衬线标题是核心风格之一 | 是 |

## image_gen 判断示例

### 必须真实生图

- 左屏服装主图。
- 右屏商品卡中的模特图、服饰图、包袋图。
- 海报外围的包、花瓶、高跟鞋、丝绸布料与陈列台。
- 统一风格的背景氛围打底图，尤其当需要摄影级质感时。

### 必须代码实现

- 品牌名、标题、副标题、价格、按钮文案。
- 搜索框、收藏按钮、底部导航、分类图标、促销标签。
- 卡片结构、间距、圆角、阴影和页面信息层级。

### 不要误判成“已完成生图”的情况

以下情况都不应算完成这个案例：

- 只搭了双手机布局和粉色渐变背景。
- 用纯 CSS 画了包和鞋的占位轮廓。
- 只保留了代码页面，没有真正生成任何服装、模特或配饰位图。
- 只在文字里写“会使用 image_gen”，但没有落地真实文件并接入。

## 提示词方向示例

这个案例的提示词应强调：

- 时尚电商展示图，而不是单件商品白底图。
- 统一的柔粉、浅紫、奶油白色板。
- 轻奢、柔和、编辑风、女性向、高级陈列摄影。
- 不要生成任何可读文字、价格、logo、状态栏或系统 UI。
- 给手机界面叠加留出足够安全区。

示例方向：

```text
为女性时尚购物 App 展示海报生成一组高端编辑风位图资产。
整体色板为 blush pink、soft lavender、cream white。
需要真实细腻的服装面料、柔和棚拍光线、轻奢商品陈列感。
不要任何可读文字、logo、水印、界面按钮、价格或系统状态栏。
```

## 字体判断示例

这个案例通常会混用两种气质：

- 标题：高反差衬线，偏时尚杂志感。
- 正文和 UI：干净的人文或几何无衬线。

可接受的近似方向：

- 标题：`Cormorant Garamond`、`Bodoni`、`Canela` 气质近似。
- UI 正文：`Inter`、`SF Pro Text`、`Manrope` 一类清爽无衬线。

如果用户是要做真机可运行原型，标题字体可以近似；如果用户要的是纯展示海报，标题字形气质优先级更高。

## 交付判断

这个案例最终交付时，应明确说明：

- 实际生成了哪些服装 / 商品 / 静物位图。
- 这些位图分别落在什么路径。
- 哪些页面结构仍由代码实现。
- 是否真的做了 16:9 海报级展示构图。

必须避免的误导性表述：

- “已经按参考图复刻”，但没有任何真实时尚素材位图。
- “已经使用 image_gen”，但只生成了背景色块。
- “已经完成 App mockup”，但服装、包袋和鞋仍是空白占位或代码近似。

## 页面级审查重点

完成后应重点检查：

- 双手机之间的比例、透视与阴影是否协调。
- 左屏主服装和右屏商品图是否属于同一套风格。
- 标题字体是否足够时尚、不会被过于普通的无衬线破坏气质。
- 粉紫色调是否柔和，不要脏灰或荧光。
- 海报外围静物是否喧宾夺主，压过手机主体。
- 生成图中是否混入伪文字、假价格、假图标或异常 UI 元素。
- 页面内部真正需要可编辑的文案是否仍由代码承担。

这个案例的价值，在于教会模型识别“真实生图优先”的场景：当用户要的是时尚电商展示图与 App mockup 时，核心任务是把高影响视觉资产真的生成出来，而不是停留在代码骨架阶段。
