# 博物馆 App 案例拆解

这个案例适合演示 `image-to-ui-skill` 在“多屏移动端 UI 还原”里的判断方式：哪些区域应该继续用代码实现，哪些图片应该保留原始素材，哪些装饰元素适合用 SVG/CSS 复刻，而不是误判成整页都要交给 image2。

## 适用场景

当用户提供一张总览参考图、外加一个页面流转视频，希望把静态设计稿做成可点击原型时，优先参考这个案例。

这个案例覆盖了三类典型页面：

- 首页：卡片、快捷入口、作品缩略图、底部导航。
- 展览列表页：分段筛选、纵向卡片列表、进入详情按钮。
- 作品详情页：大图、基础信息、分栏标签、说明文案、播放入口。

## 这个案例教什么

### 1. 多屏参考图不应停留在“静态复刻”

如果用户同时给了总览图和演示视频，默认目标不是只拼一张截图，而是把多个屏幕串成真实原型：

- 首页的展览卡片和快捷入口应可点击。
- 列表页的展览卡片应进入详情。
- 详情页的返回、收藏、底部导航应有交互反馈或跳转。

### 2. 关键视觉不一定都需要 image2

这个案例里最容易误判的是“看起来很美术”的区域。应拆开判断：

- 手机外壳、状态栏、布局、按钮、标签页、底部导航：代码实现。
- 线稿人物、花瓶、柱式图形：优先用 SVG 或轻量矢量实现。
- 柔和色块背景、浅阴影、磨砂白底感：优先用 CSS 渐变、阴影、模糊和边框实现。
- 名画缩略图与详情大图：优先使用用户提供的原图或项目内已有素材，不要让模型重画《星月夜》这类版权作品截图。

### 3. “保留原图”也是 skill 的正确决策

这个案例尤其适合强调：

- 用户已经提供了总览参考图。
- 视频里明确展示了作品缩略图和详情图。
- 这些艺术作品图像属于内容素材，不应默认重新生成。

因此，推荐做法是：

- 保留艺术作品原图。
- 只对结构性 UI 做代码还原。
- 仅在缺少小图、装饰纹理或非精确插画素材时，再补 image2 生成。

## 前期审查示例

### 整体判断

- 页面类型：艺术馆 / 博物馆导览 App，多屏移动端原型。
- 视觉风格：极简留白、米白底、细线插画、低饱和浅彩色氛围块。
- 核心布局：顶部轻导航 + 中段大卡片 / 图像 + 底部导航。
- 首屏重点：首页展览卡片、列表页分屏切换、详情页作品图和文字信息层级。
- 主要风险：字体气质、插画线条粗细、柔和背景光晕、作品图裁剪和多屏跳转逻辑。

### 元素拆分

| 区域/元素 | 建议实现方式 | 难度 | 原因 | 是否需要用户确认 |
| --- | --- | --- | --- | --- |
| 手机画布与圆角屏幕 | 代码 | 容易 | 固定比例容器即可稳定实现 | 否 |
| 标题、正文、日期、标签页 | 代码 | 容易 | 必须保持可编辑、可访问 | 否 |
| 首页主卡片结构 | 代码 | 中等 | 需兼顾圆角、留白和阴影层级 | 否 |
| 线稿人物和装饰花瓶 | SVG/CSS | 中等 | 细线装饰可矢量实现，没必要生图 | 否 |
| 背景氛围色块 | CSS | 容易 | 柔和渐变和 blur 可直接实现 | 否 |
| 名画缩略图与详情大图 | 使用原素材 | 不建议直接生成 | 属于精确内容图像，重画会失真 | 是 |
| 音频导览按钮图标 | 图标库或 SVG | 容易 | 可用 lucide 风格图标近似 | 否 |
| 字体气质匹配 | 开源字体近似 | 中等 | 几何无衬线 + 人文正文需仔细选型 | 是 |

## image2 判断示例

### 必须代码实现

- 所有可读文字。
- Tab、返回、收藏、分享、底部导航。
- 卡片布局、分隔线、标签下划线、按钮容器。
- 手机壳与整体设备构图。

### 建议保留原图

- 首页 highlights 中的画作缩略图。
- 详情页作品主图。
- 如果用户后续补充单独的艺术作品原始图片，也继续直接使用。

### 可选 image2 生成

只有在这些素材缺失时，才建议调用 image2：

- 非精确装饰性背景纹理。
- 某张占位缩略图需要统一风格补图。
- 纯装饰插图需要扩展更多同风格变体。

即使调用 image2，也要明确提示：

- 不生成任何可读文字。
- 不重画精确名画主体。
- 只生成装饰性、非版权敏感的辅助位图。

## 字体判断示例

这个案例里字体气质比“是否绝对同款”更重要：

- 标题：偏几何、轻优雅、圆润但不幼态。
- 正文：中性、清晰、略带人文感。
- 数字和 tab 文案：要干净，避免过度机械。

可接受的近似方向：

- 标题：`Manrope`、`Plus Jakarta Sans`、`Avenir Next` 风格近似。
- 正文：`Inter`、`SF Pro Text`、`system-ui` 一类中性正文。

如果是实际还原 iOS 风格原型，可优先使用系统栈，并说明字体差异主要来自平台渲染。

## 交互串联建议

默认串联路径：

1. 首页点击 `Modern Visions` 卡片进入展览列表。
2. 首页点击 `Exhibitions` 快捷入口也进入展览列表。
3. 列表页点击首张卡片进入作品详情。
4. 详情页 `Back` 返回列表。
5. 底部导航在三个页面间保持统一高亮逻辑。

如果用户没有提供更多页面：

- `Map`、`Audio Guide` 可以做成局部反馈或占位状态。
- `Favorite` 可以只做收藏态切换。
- `About / Analysis / In Context` 可以切换详情正文区域。

## 页面级审查重点

这个案例完成后，应重点检查：

- 多屏切换时，手机画布比例是否稳定，没有缩放跳变。
- 首页卡片、列表卡片和详情图的圆角半径是否一致。
- 线稿插画是否过粗、过深，避免压过正文层级。
- 浅色氛围块是否太脏、太重或颜色过饱和。
- 艺术作品图像是否被错误拉伸或过度裁剪。
- 详情页文案在窄屏下是否溢出。
- 底部导航高亮状态是否和当前页面一致。

## 最终交付示例

完成后应明确说明：

- 哪些结构完全由代码还原。
- 哪些艺术作品图片沿用原素材。
- 是否实际使用了 image2；如果没有，就写明“本案例重点是结构还原与原图保留，而非强制生图”。
- 页面是否已根据视频串成可点击原型。

这个案例的价值不在于“多生成几张图”，而在于教会模型做更准确的取舍：该代码实现的就代码实现，该保留原图的就保留原图，真的缺图时再调用 image2。
