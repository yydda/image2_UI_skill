# image_gen 入口与验真

这份文档用于回答一个关键问题：当前 skill 里真实生图到底怎么调用。答案固定为：使用系统 `imagegen` skill 的内置 `image_gen` 工具。

## 1. 唯一允许的生图入口

当前项目只允许这一条真实生图路径：

1. 触发系统 `imagegen` skill。
2. 使用默认的内置 `image_gen` 工具模式生成位图。
3. 如果图片要接入项目，把选定输出从 `$CODEX_HOME/generated_images/...` 复制到当前 workspace 的资源目录。
4. 更新页面代码或样式，让图片真实渲染在可点击 demo 中。
5. 用截图或页面检查确认资产已经显示。

不要为这个 skill 创建或调用额外的本地生图 CLI、SDK runner 或 API wrapper。

## 2. 明确禁止的旧路径

以下方式不能作为本仓库的生图入口：

- 本地生图命令或环境变量入口
- 项目内 wrapper 脚本
- 外部中转图片 API
- OpenAI SDK 自写生图脚本
- 任何需要用户 API key 的 CLI/API fallback
- 随机在线生图服务
- 用 CSS、SVG、渐变、噪点或截图近似复杂视觉后声称已经生图

如果内置 `image_gen` 工具不可用，必须停止并说明“尚未完成真实位图资产生成”。不要自动切换到任何外部 API 或付费 fallback。

## 3. 生成前记录

执行真实生图前，记录这些信息：

- 资产 id
- UI 位置和用途
- 输入图片或提示词来源
- 目标槽位尺寸
- 期望导出尺寸、比例和格式
- 是否需要透明背景、裁剪、压缩或移动端裁切
- 目标项目路径

## 4. 生图后的验真记录

真实调用 `image_gen` 后，最终回复至少列出：

| 项目 | 内容 |
| --- | --- |
| 生图工具 | `built-in-image_gen` |
| 生成资产 | 文件路径和用途 |
| 接入位置 | 哪个页面、组件或 CSS 槽位 |
| 截图验证 | 截图路径、视口尺寸、检查结果 |
| 剩余代码近似 | 哪些区域仍由 CSS/SVG/代码实现 |

如果图片只生成但没有接回页面，只能说“已生成资产”，不能说“已完成 UI 复刻闭环”。
